# Pico V1 Scope

## 목적

Pico v1은 아이가 짧게 열고, 바로 이해하고, 다시 찾아오고 싶은 영어
학습 게임 모음이다. 학습 앱처럼 설명이 많은 화면보다, 먼저 "게임답게"
느껴지는 경험을 목표로 한다.

v1은 대형 학습 플랫폼이 아니다. 한 사람이 운영 가능한 정적 웹 게임
서비스로 유지하면서, 네 가지 미니게임과 공통 보상 구조를 완성한다.

구현 태스크 목록은 `docs/v1-tasks.md`에서 관리한다.

## V1 확정 결정

- `V1-T001`: v1 출시 목표 콘텐츠 수는 게임 타입별 20개 stage로 정한다.
  틀린그림 찾기, 숨은그림 찾기, 미로 찾기, 메모리 게임을 각각 20개씩
  제공해 총 80개 playable stage를 v1 범위로 한다.
- `V1-T002`: 로그아웃 사용자는 현재 stage 안에서 얻은 점수만 볼 수
  있다. 총 포인트, 연속 방문, 영구 진행 저장은 로그인 후 제공한다.
- `V1-T003`: 첫 메모리 게임 stage는 이미지-이미지 매칭으로 시작한다.
  이미지-단어와 단어-음성 매칭은 이후 stage 난이도 상승에 사용한다.
- `V1-T004`: v1 미로는 grid 기반 엔진으로 구현한다. 이미지는 장식과
  테마 표현에 사용하고, 이동 가능 여부와 완료 판정은 grid data가
  담당한다.

## V1 구현 결과

- 틀린그림 찾기: 기존 raster stage 3개와 deterministic SVG stage 17개를
  합쳐 `contents` stage 20개를 제공한다.
- 숨은그림 찾기: `hidden_picnic_001` raster stage와 generated SVG scene
  stage 19개를 합쳐 20개를 제공한다.
- 미로 찾기: `maze_garden_001` stage와 generated grid stage 19개를 합쳐
  20개를 제공한다.
- 메모리 게임: `memory_animals_001` stage를 이미지-이미지 매칭 seed
  stage로 제공한다.
- 남은 v1 콘텐츠 확장은 메모리 게임을 20개 stage로 채우는 작업이다.
- 로그인 사용자의 총 포인트, streak, game summary, stage completion은
  `pico-progress` record에 저장한다.
- Find & Learn의 세부 foundIds는 legacy `find-learn-progress` record에도
  저장해 기존 진행 상태를 보존한다.

## 대상

- 주 사용자: 대략 4-8세 아이
- 보조 사용자: 스테이지를 골라주거나 한국어 설명을 읽어주는 보호자,
  교사, 돌봄자
- 한 번의 플레이 길이: 3-10분
- 우선 기기: 모바일 브라우저, 태블릿, 데스크톱 순서

## V1 목표

- 네 가지 게임 타입을 제공한다.
  - 틀린그림 찾기
  - 숨은그림 찾기
  - 미로 찾기
  - 메모리 게임
- 각 게임 타입은 v1 출시 기준 20개 stage를 제공한다.
- v1 전체 콘텐츠 목표는 80개 playable stage다.
- 모든 게임은 긴 설명 없이 시작할 수 있어야 한다.
- 그림, 짧은 문장, 음성을 통해 영어 단어를 자연스럽게 접하게 한다.
- 완료, 발견, 재방문을 포인트로 보상한다.
- 연속 방문을 보상해 아이가 다시 들어올 이유를 만든다.
- 로그인한 사용자는 포인트, 완료 상태, 연속 방문 기록을 저장한다.
- 로그아웃 상태에서도 현재 브라우저 세션 안에서는 플레이할 수 있다.
- 게임별 화면은 다르더라도 HUD, 점수, 음성, 완료, 이동, 로그인 경험은
  공통 구조를 사용한다.

## V1에 포함하지 않는 것

- 네이티브 모바일 앱
- 멀티플레이, 채팅, 친구 기능
- 공개 리더보드
- 광고, 결제, 구독, 현실 보상
- 보호자 대시보드나 교사용 교실 관리
- 사용자가 직접 콘텐츠를 만드는 도구
- 게임 실행 중 AI 이미지 생성
- 시험, 등급, 복잡한 커리큘럼 진도 관리

## 게임 범위

### 콘텐츠 수량

v1 콘텐츠 목표는 게임 타입별 20개 stage다. 현재 구현된 stage는 엔진,
UX, 점수, 저장 구조를 검증하는 seed set에서 출발해 순차적으로 20개까지
채운다.

| 게임 타입 | v1 목표 stage 수 | 현재 stage 수 |
| --- | ---: | ---: |
| 틀린그림 찾기 | 20 | 20 |
| 숨은그림 찾기 | 20 | 20 |
| 미로 찾기 | 20 | 20 |
| 메모리 게임 | 20 | 1 |
| 합계 | 80 | 61 |

### 틀린그림 찾기

현재 구현 이름: `Find & Learn`

두 그림을 비교해 다른 부분을 찾는다. 정답을 누르면 점수를 얻고 영어
피드백을 듣는다. 사물을 누르면 영어 단어 카드와 예문을 볼 수 있다.

v1 필수 조건:

- 두 그림은 모든 지원 화면에서 좌우로 나란히 보인다.
- 정답을 누를 수 있는 쪽은 stage data에 명확히 기록한다.
- 힌트, 리셋, 진행률, 점수, 완료 흐름을 지원한다.
- 정답/오답 피드백은 짧은 영어 음성으로 제공한다.
- 한국어 보조 텍스트는 제공하되, 게임 흐름은 시각적으로 이해 가능해야
  한다.

### 숨은그림 찾기

하나의 장면 이미지 안에서 목표 사물을 찾는다. 목표 목록은 영어 단어와
작은 상태 표시로 제공한다.

v1 필수 조건:

- 한 장의 큰 장면 이미지를 보여준다.
- 난이도에 따라 3-8개의 목표 사물을 제공한다.
- 각 목표는 영어 단어, 한국어 뜻, 선택적 발음 표기, 예문, 이미지 기준
  클릭 영역을 가진다.
- 찾은 목표는 장면 안에 표시하고 목표 목록에서도 완료 처리한다.
- 힌트는 남은 목표 하나를 강조하지만, 즉시 정답 처리하지 않는다.
- 모든 목표를 찾으면 완료된다.

### 미로 찾기

캐릭터나 마커를 출발점에서 목표 지점까지 이동시킨다. 미로 안에는 선택
수집 단어 아이템을 둘 수 있다.

v1 필수 조건:

- 모바일 터치와 데스크톱 포인터 입력을 지원한다.
- 출발점과 도착점이 명확해야 한다.
- 어린아이에게 가혹한 실패 루프를 만들지 않는다. 잘못된 길은 부드럽게
  되돌리거나 안내한다.
- 선택 수집 아이템은 작은 보너스 포인트를 주고 영어 단어를 말한다.
- 도착점에 도달하면 완료된다.
- 미로 데이터는 grid, 출발점, 도착점, 장애물, 선택 수집 아이템을 표현할
  수 있어야 한다.

### 메모리 게임

카드를 뒤집어 같은 짝을 찾는다. 스테이지에 따라 이미지-이미지,
이미지-단어, 단어-음성 매칭을 사용할 수 있다.

첫 v1 stage는 어린아이가 규칙을 바로 이해할 수 있도록 이미지-이미지
매칭을 사용한다.

v1 필수 조건:

- 4, 6, 8, 12쌍 카드 구성을 지원한다.
- 카드 레이아웃은 모바일 터치에 충분히 커야 한다.
- 카드를 열거나 짝을 맞추면 영어 단어를 말한다.
- 짝을 맞출 때마다 점수를 주고, 완료 보너스를 준다.
- 시도 횟수를 기록해 나중에 난이도 조정에 사용할 수 있게 한다.
- 틀린 짝 피드백은 짧고 부드럽게 처리한다.

## 공통 게임 UX

v1의 모든 게임은 아래 화면 모델을 공유한다.

- 게임 선택 화면 URL은 `/`이다.
- 하나의 stage를 플레이하는 게임 화면이 있다.
- 게임 화면에는 `Games` 버튼이 있어 선택 화면으로 돌아갈 수 있다.
- 버튼은 `Games`, `Hint`, `Reset`, `Log in`, `Log out`처럼 눈에 보이는
  역할 라벨을 가진다.
- stage 제목, 진행률, 점수, 완료 상태는 아이용 HUD에 표시한다.
- 영어 단어, 예문, 한국어 뜻, 음성 다시 듣기는 간단한 학습 패널에
  표시한다.
- 완료 화면은 다음 stage가 있으면 다음으로 이동할 수 있어야 한다.
- 터치 대상은 아이 손가락으로 누르기 충분히 커야 한다.
- 게임 화면에 긴 사용 설명을 넣지 않는다.
- 리셋은 내비게이션이나 계정 버튼과 시각적으로 구분한다.

## 꼭 필요한 V1 아이디어

### 오늘의 플레이 이유

첫 화면에 "오늘 한 판" 느낌의 진입점을 둔다. 아이는 무엇을 해야 할지
고민하지 않고 바로 한 stage를 시작할 수 있어야 한다.

필수 동작:

- v1 현재 구현은 가장 쉬운 seed stage를 빠른 시작으로 추천한다.
- 출시 콘텐츠가 80개로 확장되면 완료하지 않은 쉬운 stage를 우선
  추천한다.
- 추천은 복잡한 알고리즘보다 아이가 바로 시작할 수 있는 단순함을 우선한다.

### 부드러운 도움

아이용 게임은 막혔을 때 떠나는 순간이 빠르다. 힌트는 점수보다 플레이
지속을 우선한다.

필수 동작:

- 일정 시간 동안 아무 정답이 없으면 힌트 버튼을 시각적으로 강조할 수
  있다.
- 힌트를 써도 stage 완료는 가능해야 한다.
- 힌트 사용은 기록하되 v1에서는 큰 벌점으로 만들지 않는다.

### 작은 축하

정답, 매칭, 완료 순간에는 짧은 긍정 피드백이 있어야 한다. 큰 보상
시스템보다 즉각적인 반응이 중요하다.

필수 동작:

- 정답이나 매칭에는 마커, 색 변화, 짧은 음성 중 최소 하나가 반응한다.
- stage 완료 시 점수와 다음 행동을 명확히 보여준다.
- 완료 후 선택지는 `Next`, `Home`, `Stay`처럼 짧고 분명해야 한다.

## 포인트와 보상

포인트는 아이를 압박하기 위한 점수가 아니라, "찾았다", "끝냈다",
"다시 왔다"를 칭찬하는 장치다.

### 기본 포인트

- 틀린그림 정답 1개: 100점
- 숨은그림 목표 1개: 100점
- 미로 완료: 300점
- 미로 수집 아이템 1개: 50점
- 메모리 카드 짝 1개: 100점
- stage 완료 보너스: 200점

규칙:

- 완료 보너스는 사용자별 stage당 한 번만 지급한다.
- 이미 완료한 stage를 다시 플레이할 수는 있지만 총점이 반복해서
  무한히 늘어나면 안 된다.
- 점수는 stage별 점수와 사용자 총점으로 저장한다.
- 로그아웃 상태에서는 현재 stage 안에서 얻은 점수만 보여준다.
- 영구 포인트 저장은 로그인 사용자에게만 제공한다.

### 연속 방문 보상

v1은 연속 방문을 보상한다.

방문 인정 기준:

- 하루에 하나 이상의 stage를 완료하거나 게임 안에서 1점 이상 획득하면
  그날 방문으로 인정한다.
- 기준 날짜는 사용자의 로컬 날짜를 사용한다.
- 하루를 놓치면 다음 인정일에 streak는 1부터 다시 시작한다.

권장 보상:

- 1일차: 50점
- 2일차: 75점
- 3일차: 100점
- 4일차: 125점
- 5일차 이상: 매일 150점

규칙:

- 연속 방문 보상은 하루 한 번만 지급한다.
- 로그인 사용자는 게임 선택 화면에서 현재 streak를 볼 수 있다.
- 로그아웃 사용자는 설명이 복잡해지면 streak UI를 숨긴다.

## 난이도

각 stage는 난이도를 가진다.

- Level 1: 매우 쉬움, 3-4개 목표나 카드쌍, 넓은 클릭 영역, 타이머 없음
- Level 2: 쉬움, 5-8개 목표나 카드쌍, 적당한 시각 밀도
- Level 3: 보통, 8-12개 목표나 카드쌍, 더 많은 방해 요소

v1에는 카운트다운 타이머를 넣지 않는다. 플레이 시간은 내부 기록으로
남길 수 있지만 아이에게 시간 압박을 주지 않는다.

## 콘텐츠 방향

v1은 아이에게 익숙하고 안전한 장면을 우선한다.

- 아이 방
- 놀이터
- 소풍
- 교실
- 부엌
- 동물원이나 동물 공원
- 장난감 방
- 공원이나 정원

영어 단어는 구체 명사와 쉬운 행동을 우선한다.

- 명사: ball, book, apple, cat, robot, chair, tree
- 색: red, blue, green, yellow
- 행동: sitting, running, jumping, eating, sleeping
- 짧은 문장: "The cat is sleeping.", "The ball is red."

## 데이터 모델 방향

v1은 게임별 registry가 아니라 공통 stage registry로 확장할 수 있어야
한다. 각 stage는 아래 공통 필드를 가진다.

```js
{
  id: "hidden_playground_001",
  gameType: "hidden_objects",
  title: "Playground Hunt",
  titleKo: "놀이터 찾기",
  level: 1,
  theme: "playground",
  estimatedMinutes: 3,
  previewImage: "/games/hidden_playground_001/preview.png",
  points: {
    completionBonus: 200
  }
}
```

게임별 데이터는 공통 필드를 확장한다.

- 틀린그림 찾기: panels, differences, objects
- 숨은그림 찾기: scene image, target objects, hit areas
- 미로 찾기: grid, start, goal, obstacles, collectibles, optional theme
  image
- 메모리 게임: card pairs, card faces, match mode

## 저장 방향

v1은 ohmesh에 사용자별 Pico 진행 record 하나를 저장하는 방향으로 간다.

저장해야 하는 정보:

- 총 포인트
- 연속 방문 상태
- 게임별 완료 요약
- stage별 최고 점수와 완료 상태
- stage별 found IDs, matched IDs, maze completion
- 마지막 방문 인정 날짜
- 마지막 업데이트 시각

초안 형태:

```js
{
  version: 2,
  totalPoints: 1250,
  streak: {
    current: 3,
    longest: 5,
    lastQualifiedDate: "2026-05-30",
    lastRewardDate: "2026-05-30"
  },
  games: {
    spot_the_difference: {
      completedStages: 2,
      points: 800
    },
    hidden_objects: {
      completedStages: 1,
      points: 350
    }
  },
  stages: {
    hidden_playground_001: {
      gameType: "hidden_objects",
      completed: true,
      score: 350,
      foundIds: ["ball", "kite", "apple"],
      completedAt: "2026-05-30T00:00:00.000Z",
      updatedAt: "2026-05-30T00:00:00.000Z"
    }
  }
}
```

기존 Find & Learn progress record는 v1 저장 구조 도입 시 legacy shape로
읽거나 migration한다.

## V1 출시 체크리스트

- 게임 선택 화면이 네 가지 게임 타입의 stage를 보여준다.
- 각 게임 타입마다 20개씩, 총 80개의 완성도 있는 playable stage가 있다.
- 각 게임은 모바일 터치 입력을 지원한다.
- 각 게임은 가능한 범위에서 키보드 또는 포인터 입력도 지원한다.
- 공통 HUD는 stage 제목, 진행률, 점수, 명확한 액션 버튼을 보여준다.
- 공통 완료 화면은 다음 stage와 게임 선택 화면 이동을 지원한다.
- 포인트가 일관되게 지급되고 로그인 사용자에게 저장된다.
- 연속 방문 보상이 하루 한 번 지급된다.
- 로그아웃 상태에서도 플레이할 수 있다.
- `npm run lint`와 `npm run build`가 통과한다.
- README와 `docs/spec.md`가 이 v1 범위 문서를 연결한다.
